El cometido de este flujo es notificar al cajero que generó un QR cuando se realiza un pago por parte del cliente.
Como funciona hoy en día notification-service y pago movil
Primero hay que registrar el token. Hay que llamar a endpoint de Notification-service para registrar el user_device
user_id => usuario loggeado en la app (el que va a recibir la push)

token => token usado para enviar la push Notification

platform => android, ios

app_short_name => pm, mpos. Por defecto es pm (Pago Movil)

Para enviar una push, hay que llamar a `send_push_notification` del servicio notification-service, indicando
user_id => usuario al que le llegará la push

data => datos que se le quiera mandar en la push

message => mensaje que aparecerá en la push

app_short_name => de donde viene (pm, mpos)

notification-service, en función del user_id y la app_short_name, busca el token asociado y envía en forma asíncrona la push con esos datos y el mensaje
