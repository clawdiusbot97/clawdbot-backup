Este documento detalla el plan de trabajo y estimación de tareas para reorganizar los portales actuales de la suite de comercios, con el objetivo de mejorar la mantenibilidad, escalabilidad y experiencia de desarrollo.
🔁 Contexto actual
commerce-suitees el portal legado que contiene tanto vistas de comercios como de Boca Web.

Boca Web se va a migrarsin rediseñoa un nuevo portal llamadoepos-portal.

Se creócommerce-portal, un nuevo portal que reemplazará gradualmente acommerce-suite.

🎯 Objetivos
Migrar progresivamente las vistas decommerce-suiteacommerce-portal.

Desdecommerce-portal, redireccionar inicialmente haciacommerce-suitepara acceder a vistas aún no migradas.

Actualizar e implementar nuevas vistas directamente encommerce-portal.

Apagarcommerce-suiteuna vez que toda la funcionalidad esté en el nuevo portal.

Migrar Boca Web al nuevoepos-portaltal como está, separándolo definitivamente decommerce-suite

🧭 Plan de transición
Desarrollar la estructura base decommerce-portalyepos-portal.

Montar las vistas nuevas de comercio directamente encommerce-portal.

Para vistas aún no migradas, usarcommerce-portalcomo entrypoint, redireccionando haciacommerce-suite.

Migrar Boca Web completa aepos-portalsin rediseño.

Desacoplar y apagarcommerce-suiteuna vez que todos los módulos estén migrados y validados.

Las horas estimadas son:262
Observaciones adicionales
Este plan no contiene tareas de rediseño, solo de arquitectura
