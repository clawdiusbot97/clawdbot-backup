Prueba con implementación de joins:
Se prueba cambiar includes por joins para algunos campos de la consulta, para ver el comportamiento.

Para estos casos se lanza una query por cada transacción, buscando commerce_branches, payment_processors y devices, por lo que se generan problemas de tipo n+1 por cada transacción.
En este caso de prueba, se hacen tres queries por cada transacción, una por cada elemento dentro del joins, por lo que hace que el flujo demore mucho más en terminar, siendo que se hacen miles de queries para llegar a terminar.
El flujo completo, con un solo batch de transacciones, termina con un tiempo de más de 5 minutos
