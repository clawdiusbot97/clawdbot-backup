The aim of this document is to provide a guide for the proper configuration of an IPA user in the staging environment, ensuring fast and seamless access to all servers.
Requirements
For connecting and navigating between staging machines, we will perform SSH key forwarding. This will allow us to use our own SSH key to make jumps from the development bastion. To use key forwarding, we need to first add our SSH key to the SSH agent using the following command:
ssh-add -K ~/.ssh/id_rsa (in macOS)

ssh-add ~/.ssh/id_rsa (in Linux)

Since access to this bastion will be quite frequent, to avoid entering a password, we will add our public key to the authorized keys on this machine:
Enter the bastion usingssh bastion-bancardand our password.

Create (if it doesn't exist) the folder~/.sshwith the commandmkdir ~/.ssh.

Create (if it doesn't exist) the file~/.ssh/authorized_keysusing the commandvim ~/.ssh/authorized_keysand add our public SSH key to it, which can be found in the file~/.ssh/id_rsa.pubon our machine.

We modify the access permissions of the~/.sshfolder using the commandchmod 700 ~/.sshand of the~/.ssh/authorized_keysfile on the bastion using the commandchmod 600 ~/.ssh/authorized_keys.

By following these steps, we should now be able to SSH into the development bastion using the commandssh bastion-bancard, without requiring a password.

Return to the development bastion and repeat steps 1 to 4 for the rest of the servers where dockerized services are located.
