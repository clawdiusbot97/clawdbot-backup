Proceso delbuild
Comienzo: Push a GitHub
Al pushear nuestros cambios a GitHub y crear la PR, circle CI ejecutará losjobs. Estos se encuentran definidos en.circleci/config.ymlpara cualquier servicio. Hay 2 tipos de jobs definidos: tests y provide. Analizaremos en profundidad sus contenidos y forma de ejecución, pero introduciremos una convención general:testsse ejecutará siempre, para todo tipo de push a GitHub.Providecorrerá siempre y cuando la branch a la que estemos pusheando seamaster, develop, sandboxo que su nombre comience confeature/(la branch de una nueva iniciativa) oprovide/(así forzando que se corra el provide).
