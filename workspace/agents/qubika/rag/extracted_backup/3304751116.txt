Requisitos:
Ruby >= 3.1

MySQL Client (Descargaraquipara macOS)

Docker Desktop (Descargaraqui)

Permiso de lectura en los repositorios de Bancard

Clave ssh (ed25519) registrada en GitHub

Creación del Repositorio en Git Hub:
Crear un nuevo repositorio en la organizaciónbancardsa.

Deja el repositorio vacío, no uses ninguna plantilla.

Configura las reglas del repositorio de la siguiente manera (Settings/Rulesets → Branch Rules):
Main:
Target branch es simplemente 'main'

Tildar 'Restrict Deletions'

Tildar 'Require a pull request before merging' con las opciones:
2RequiredApprovals

Dismiss stale pull request approvals when new commits are pushed

Require approval of the most recent reviewable push

Require conversation resolution before merging

Tildar 'Block Force Pushes'

Más tarde volveremos para completar las reglas, una vez corramos las GH actions por primera vez

Develop:
Target branch es simplemente 'develop'

Se repiten todas las reglas de master

Ramas Feature:
Target branch serán todas aquellas de la forma 'feature/*'

Se repiten todas las reglas de master y develop

Configurar las reglas generales (Settings/General)
La rama base debe ser develop

En Pull Requests habilitar la opción 'Automatically delete head branches'

Creación del proyecto - Angus:
En tu propio sistema corre el comandogem install angus

En el directorio donde quieras tu nuevo proyecto corre el comandoangus new my_app

Distinto de Rails, las apps de Angus no se inicializan como repositorios de git
Corre el comandogit initen la carpeta del proyecto

Corregit remote add my_github_repository

El contenido disponible ahora mismo será la base del resto del proyecto. Puedes crear las ramasmain,developy la primera ramafeature/project-setup, y pushear las referencias a tu repositorio.

Todas las PRs del proceso de setup deben hacerse hacia project-setup

Creación del proyecto - Primer Endpoint:
Si bien Angus tiene una funcionalidad de Scaffold para fabricar endpoints completos, es recomendable fabricarlos a mano siguiendoesta guía.
Corre el comandoangus servery desde Postman intenta alcanzar el endpoint de documentación y tu nuevo servicio.
Doc:localhost:XXXX/my_app/doc/0.1
Creación del proyecto - Dockerización:
A partir de este punto se requiere tener instalado docker y docker-compose, o Docker Desktop que incluye ambos.
docker-compose.yml:

Define los contenedores, sus versiones y cómo interactúan entre si. Recuerda definir en el service qué puerto vas a utilizar, en la propiedadcommand.
Dependiendo de los requisitos de tu proyecto puede que necesites un contenedor de sidekiq.
Dockerfile:

Para todos los servicios utilizamos jruby en los servidores on premise de Bancard. Si actualizas la versión de Ruby más allá del 3.1 es imperativo revisar qué version de jruby le corresponde y actualizarla
local-entrypoint.sh

Es el entrypoint que utilizaremos para levantar el contenedor 'service'
wait-for-it.sh:

Este archivo es una rutina que permite al contenedor de la DB inicializarse por completo antes de enviarle una primera conexión
En principio con estos 4 archivos ya sería suficiente para levantar el proyecto completo, pero para ahorrarnos el manejo directo de nuestros contenedores el tech core preparó una serie de scripts que hacen casi todo por nosotros.
bin/run

Inicializa todos los contenedores, con todas las variables y reemplaza todos los archivos que se van a requerir.
--pruneborrará los archivos de configuración viejos para reemplazarlos por el .sample más reciente
--prune-forceademás limpiará los archivos de redis y mysql
bin/destroy

Da de baja los contenedores vigentes del proyecto
--db remueve la data de mysql
--redis remueve la data de redis
-a/--all remueve ambas
bin/restart

Reinicia el entorno vigente y todos sus contenedores con configuraciones idéntivas
bin/terminal

Da acceso a la terminal deservice
A estas alturas el objetivo es que bin/run levante todos los contenedores y, acorde al puerto que hayas definido en tus configuraciones, puedas hacer un llamado al primer endpoint que configuraste en el paso anterior.
