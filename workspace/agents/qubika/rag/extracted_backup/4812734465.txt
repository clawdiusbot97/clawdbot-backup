Consideraciones Generales
El proceso 3DS en cybersource está dividido en 5 pasos, los cuales son nuestra guía para el armado de la estimación:
Setup Service:llamado a la API de cybersource enviando datos de la tarjeta para obtener la URL y el JWT para el DDC.

Device Data Collection:llamado a la DDC URL, recolección de los datos del navegador en iframe invisible.

Payer Authentication Check Enrollment Service:(ex lookup) llamado a la API de cybersource para conocer el nivel de riesgo de la operación:
- Frictionless success (riesgo bajo)
- Challenge required (riesgo moderado)
- Frictionless failure or decline (riesgo alto)

Step-Up iframe:iframe visible donde el usuario completa el challenge para validar su identidad.

Payer Authentication Validation Service:verificación de que el usuario validó exitosamente su identidad (solo si ocurrió un challenge).

Tpago
El flujo 3DS en Tpago está presente en los flujos:
Pago Ocasional Sin Login (en conjunto con Kount. Si kount está activo, solo si kount aprueba. Si kount está inactivo, se realiza 3DS)

Pago Ocasional con Login (Formulario de tarjeta)

Pago con tarjeta catastrada

Flujo stripe (Configurable, apagado al momento de escribir este doc)
