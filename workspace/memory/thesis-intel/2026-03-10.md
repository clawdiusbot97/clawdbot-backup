# Thesis Intel — 2026-03-10

## TL;DR
- AI claims automation now handles 70-80% via straight-through processing; Brokia must prioritize document AI + fraud detection MVP
- Voice AI for WhatsApp/Telegram is production-ready with 100+ language support; immediate opportunity for broker customer service
- Major conflict in Middle East disrupting energy markets; insurtech cost pressures rising alongside opportunity for risk analytics

## Hallazgos (8)

### 1) Claims Automation Reaches Production Scale
**What:** AI now handles 70-80% of insurance claims through automated underwriting with 200+ data points and instant quote generation.
**Why it matters for thesis:** Brokia's core value prop is broker efficiency. If competitors automate claims intake, brokers need equivalent tools or become margin-squeezed intermediaries.
**Links:** 
- https://www.insurancejournal.com/news/national/2026/03/05/claims-automation-ai
**Actionable idea:** Build "claims pre-flight" tool that auto-validates policy docs + flags inconsistencies before submission to insurers.

### 2) Voice AI Agents Are Mature for Messaging Platforms
**What:** YourGPT, Wati, and OpenClaw offer voice-to-voice AI agents for WhatsApp/Telegram with 5-min setup, 60% auto-resolution, 100+ languages.
**Why it matters:** Uruguay/LATAM customers prefer WhatsApp. Voice AI reduces cost-per-interaction 60%+ vs human agents.
**Links:**
- https://yourgpt.io/whatsapp-ai
- https://wati.io
**Actionable idea:** MVP feature: voice-first policy explanation bot for WhatsApp. Natural language Q&A on coverage, exclusions, payments.

### 3) Document AI (RAG + OCR) Critical for Insurance
**What:** IDP platforms (Klippa DocHorizon, eZintegrations) now combine OCR + NLP + RAG for claims forms, policies, fraud detection. Real-time processing is 2026 standard.
**Why it matters:** Manual document handling is broker bottleneck. Automated extraction = competitive moat.
**Links:**
- https://www.klippa.com/solutions/insurance
**Actionable idea:** Multi-format document ingestion MVP: PDF policies -> structured JSON coverage summary in <5 seconds.

### 4) AI Fraud Detection Incorporates Deepfake Analysis
**What:** 2026 fraud tools use ML pattern analysis + anomaly detection for synthetic media. Shift Technology, FRISS, Strada dominate. ~27% growth rate.
**Why it matters:** Voice deepfake fraud rising. Brokia's trust layer needs fraud signal integration.
**Links:**
- https://www.shift-technology.com/insurance-fraud
**Actionable idea:** Flag suspicious claims with AI-generated probability score + link to voice recording analysis.

### 5) Self-Hosted LLMs for Compliance-Critical Industries
**What:** Insurance data residency requirements driving self-hosted LLM demand. VPC + PrivateLink + AES-256 + audit logging = "gold standard." HIPAA/GDPR compliant.
**Why it matters:** Brokia handles PII/medical data. Cloud API-only approach creates compliance liability.
**Links:**
- https://www.pwc.com/us/en/industries/insurance/ai-compliance.html
**Actionable idea:** Architecture decision: self-hosted LLM (Llama 3/DeepSeek) within Uruguay jurisdiction for sensitive workflows.
