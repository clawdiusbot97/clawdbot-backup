## 1) Estructura mínima recomendada (contenido)

> Nota: esta estructura sirve tanto para anteproyecto como para documentos intermedios. Para entrega final, agregar los componentes adicionales indicados en la sección 3.

### 1.1 Portada
- Título
- Autores
- Institución
- Fecha

### 1.2 Resumen / Abstract (máx. 400 palabras)
- Contexto
- Problema
- Objetivo
- Alcance
- Metodología de validación (en 3–5 líneas)

### 1.3 Palabras clave
- 4–8 términos

### 1.4 Introducción
- Contexto del dominio
- Motivación
- Alcance del documento

### 1.5 Descripción del sistema actual (AS‑IS)
- Actores (incluyendo Broker)
- Relaciones
- Procesos principales
- Fricciones observadas

### 1.6 Problema y justificación
- Problema general
- Delimitación: por qué el problema es relevante
- Evidencia preliminar (sin prometer resultados)

### 1.7 Objetivos
- Objetivo general
- Objetivos específicos (medibles)

### 1.8 Alcance y no‑alcance
- Qué entra
- Qué no entra
- Supuestos
- Restricciones

### 1.9 Metodología de validación (síntesis)
- Qué evidencia se recolectará (entrevistas/encuesta)
- Métricas base
- Umbrales de decisión

### 1.10 Riesgos
- Metodológicos
- Operativos
- Dependencias de terceros

### 1.11 Plan de trabajo (alto nivel)
- Etapas
- Hitos
- Entregables

### 1.12 Referencias bibliográficas (formato IEEE)

### 1.13 Anexos
- Instrumentos
- Tablas
- Mapa conceptual

## 2) Checklist académico (coherencia)

- [ ] El documento deja claro **qué problema** se estudia y **por qué** importa.
- [ ] El problema está **delimitado** (qué sí / qué no).
- [ ] Los objetivos específicos se pueden **medir**.
- [ ] La evidencia propuesta es **trazable** (qué se recolecta, a quién, cuándo).
- [ ] Se reconocen dependencias externas (p. ej., broker/compañía) para evitar atribución incorrecta.
- [ ] No se confunde problema con solución (evitar describir componentes o arquitectura).

---

## 3) Checklist formal (ORT — formato)

> Este bloque es una lista de verificación práctica (antes de exportar a PDF).

### 3.1 Resumen / Abstract
- [ ] Abstract ≤ 400 palabras.

### 3.2 Referencias
- [ ] Referencias en formato IEEE.

### 3.3 PDF y metadatos
- [ ] El documento se exporta a PDF.
- [ ] Propiedades del PDF coherentes con la portada (título, autores, palabras clave).

### 3.4 Formato general
- [ ] Tamaño de letra: 12 pt (consistente).
- [ ] Márgenes: ≥ 2.5 cm.
- [ ] Numeración: secuencia única; número abajo a la derecha.
- [ ] Sin encabezado/pie (excepto número de página).
