## 2. Estructura de MVP-0

### Qué medimos
| Variable | Definición | Por qué importa |
|----------|-----------|-----------------|
| Tiempo total cotización | Desde que cliente pide cotización hasta que corredor presenta opciones | Dolor principal identificado |
| Tiempo por aseguradora | Tiempo dedicado a cada aseguradora contactada | Identifica escalabilidad del problema |
| Tasa de error | Cotizaciones con datos incorrectos / devueltas por aseguradora | Calidad del proceso actual |
| Tasa de conversión | Cotizaciones enviadas / Pólizas cerradas | Eficiencia del corredor |

### Cómo lo medimos
- **Herramienta:** Google Sheets compartida + cronómetro manual (celular)
- **Proceso:**
  1. Cliente solicita cotización → JM inicia timer
  2. Cada interacción con aseguradora se registra (nombre aseguradora, tiempo dedicado)
  3. Cotización enviada al cliente → stop timer
  4. Resultado anotado (cerró/no cerró, monto, tiempo hasta respuesta)

### Qué variable define éxito
**Hipótesis de éxito:** Si el corredor promedio dedica > 6 horas semanales (15% de jornada) a cotización manual Y reportaría "bastante" o "mucho" el dolor de esta tarea → MVP-0 validado, procedemos a MVP-1.

**Hipótesis de fracaso:** Si < 3 horas semanales O el corredor dice "no es un problema grave" → pivot o drop.

## 3. Decisión Forzada de Modelo de Negocio

### Elección: B) Red de corredores paga (SaaS B2B)

#### Por qué NO las otras:
- **A) Corredor individual:** Bajo poder adquisitivo, precio sensible, churn alto, costo de adquisición no rentable.
- **C) Aseguradora paga:** Riesgo de convertirse en "empleado" de aseguradora; dependencia de unos pocos clientes; pérdida de independencia de producto.
- **D) Híbrido:** Complejidad prematura. Elegir una, dominarla, luego expandir.

#### Justificación B:
| Dimensión | Análisis |
|-----------|----------|
| **Incentivo económico** | Red de corredores (ej: "> 10 corredores") tiene presupuesto de tecnología; pain compartido; decisión centralizada; ROI fácil de calcular (horas ahorradas × costo hora × N corredores) |
| **Escalabilidad** | Acordar con 1 red = 10-50 corredores de una. Customer acquisition cost distribuido. Base para expansion regional (AR/CL/MX con mismas redes multinacionales) |
| **Riesgo estratégico** | Bajo: no dependencia de aseguradoras; margen de maniobra de producto; posibilidad pivote hacia C más adelante si tiene sentido |

**Precio tentativo:** $50-100/corredor/mes cobrado a la red. Primer cliente: red de 10 corredores = $500-1000 MRR inicial validado.
